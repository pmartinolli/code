Comment ça marche ?

Un code JavaScript exécuté dans le navigateur de l'utilisateur interroge Wikidata et récupère les données des QIDs demandés. Il les présente dans un tableau avec des filtres basés sur une liste d'autorité de certains Main subjects (P921).

Ce programme a été vibe coded avec Claude Opus 4.1 d'abord puis avec ChatGPT 4.x en août 2025. Les données de Wikidata ont été améliorées pour répondre aux besoins du programme, principalement avec P921.

Curation

Liste du bibliothécaire (370+)

Le tableau des revues existait sous la forme d'un document PDF mis à jour annuellement sur le guide histoire des Bibliothèques UdeM. Il a été conçu par Marie-Ève Ménard et amélioré par Émilie Dalpé. Ce tableau a été versé dans OpenRefine et les QIDs des revues ont été alignées (complètement). Quelques contributions ont été faites pour améliorer les données des revues. Les revues ont été ajoutées à la liste de suivi (Watchlist) pour surveiller les contributions futures et le vandalisme.

Liste «histoire» du ScimagoJR (1800+)

Les données ont été téléchargées depuis ScimagoJR > Arts & Humanities > History : Download data. Puis elles ont été versées dans OpenRefine et les QIDs des revues ont été alignés (partiellement).

Liste «histoire» du DOAJ (1200+)

Les données ont été téléchargées depuis le CSV data dump du DOAJ. Puis elles ont été versées dans OpenRefine, seules les données history ont été sélectionnées dans la colonne Subjects. Les QIDs des revues ont été alignés (partiellement).

Liste «histoire» du JCR (1400+)

Les données ont été téléchargées depuis le Journal Citation Reports (de Web of Science) > Group History & Archeology. Puis elles ont été versées dans OpenRefine, les 9 catégories du groupe ont été sélectionnées. Les QIDs des revues ont été alignés (partiellement).
